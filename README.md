# contact-ayx-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态形式存储，不依赖于特定域名或网站环境，可作为通用资源进行查阅与分发。

## 项目简介

这是一个以 HTML 页面为主要内容的归档仓库。页面内容涵盖信息展示、说明文档、参考指南等类型，均以独立 HTML 文件形式存放，方便直接打开或嵌入其他项目使用。

仓库不针对任何具体网站或服务，仅作为静态页面的集中存储与版本管理用途。

## 目录说明

```
contact-ayx-guide/
├── index.html          # 仓库索引页（可选）
├── pages/              # 归档的 HTML 页面存放目录
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── README.md           # 本文件
```

- `pages/` 目录用于存放所有归档的 HTML 页面文件。
- 根目录下的 `index.html` 为可选文件，用于提供页面列表或导航功能。
- 各页面命名应尽量体现其内容主题，便于识别与引用。

## 页面归档说明

- 所有页面均为纯 HTML 格式，不依赖外部数据库或后端服务。
- 页面中引用的资源（如 CSS、JavaScript、图片等）应尽量内联或使用相对路径，以保持独立性。
- 每个页面建议保持内容简洁、结构清晰，避免过度依赖外部链接。
- 页面文件命名请使用英文小写字母、数字及连字符，避免使用空格或特殊字符。

## 维护说明

- 本仓库由维护者定期更新，新增或修改页面时请遵循现有目录结构。
- 提交前请确保 HTML 文件语法正确，无明显渲染错误。
- 不建议在页面中包含外部跟踪脚本、广告代码或动态内容加载。
- 如需删除或替换页面，请及时更新索引文件（如有）。
- 欢迎通过 Issue 或 Pull Request 提出建议或报告问题。

## 使用方式

你可以直接通过以下方式使用本仓库中的页面：

- 克隆仓库到本地，直接在浏览器中打开 HTML 文件。
- 使用 GitHub Pages 或其他静态托管服务发布整个仓库。
- 将单个页面嵌入到其他项目或文档中。
